# Перенос раздела «Заявки» в проект (VS Code)

Этот пакет содержит **раздел «Заявки»** платформы «Функция»: список заявок,
карточка заявки (с вкладками) и мастер создания заявки.

Раздел работает **внутри оболочки приложения** (`Shell` — сайдбар + шапка),
поэтому файлов больше, чем у главной.

---

## 1. Что куда класть

Скопируйте файлы в корень проекта (поверх старых, с заменой):

### Сам раздел — это и правим
| Файл | Что внутри |
|------|-----------|
| `screens_orders.jsx` | `Orders` (список + фильтры), `OrderDetail` (карточка с вкладками: Описание / Проектировщики / Коммуникации / Замечания / Файлы), `OrderNew` (мастер из 5 шагов с живым предпросмотром) |

### Данные
| Файл | Что внутри |
|------|-----------|
| `data.jsx` | `window.DATA` → список заявок `ORDERS`, статусы `STATUS`, расшифровки кодов разделов/стадий |

### Нужно, чтобы раздел отрисовался
> В вашем проекте эти файлы уже есть (это та же база) — заменяйте их только при необходимости.

| Файл | Зачем |
|------|------|
| `shell.jsx` | Оболочка `Shell` (сайдбар + шапка) и компонент `Avatar` |
| `icons.jsx` | Иконки `Icon` + логотип `FuncMark` |
| `screens_catalog.jsx` | Отсюда берётся поле поиска **`PromptSearch`** (строка «Опишите задачу…» с загрузкой файлов над списком заявок). Без него поиск не отрисуется |

### Картинки превью объектов  ⚠️ обязательно
| Файл | Зачем |
|------|------|
| `assets/hero-industrial.png` | Превью промышленного объекта (карточка + шапка заявки) |
| `assets/hero-commercial.png` | Превью коммерческого объекта |
| `assets/hero-private.png` | Превью частного дома |

Папку `assets/` кладите рядом — пути в коде указаны как `assets/hero-….png`.

### Стили и шрифты
| Файл | Зачем |
|------|------|
| `styles.css` | Глобальные переменные + импорт `components.css` и `layouts.css` |
| `components.css` | Кнопки, карточки, бейджи, чипы, выпадашки, табы — вся «начинка» раздела |
| `layouts.css` | Сетки, оболочка, таймлайн, степпер |
| `fonts/` | Шрифт Inter (woff2) |

## 2. Подключение в HTML

В `<head>`:
```html
<link rel="stylesheet" href="styles.css" />
```
(`components.css` и `layouts.css` подтянутся сами через `@import` внутри `styles.css`.)

Перед `</body>` — скрипты. Порядок важен только в одном: `icons` идёт раньше
`shell`, а `shell`, `data`, `screens_catalog` — раньше `screens_orders`:
```html
<script type="text/babel" src="icons.jsx"></script>
<script type="text/babel" src="data.jsx"></script>
<script type="text/babel" src="shell.jsx"></script>
<script type="text/babel" src="screens_catalog.jsx"></script>
<script type="text/babel" src="screens_orders.jsx"></script>
```

Экраны раздела регистрируются как `window.Orders`, `window.OrderDetail`,
`window.OrderNew`. В роутере они открываются внутри оболочки:
```jsx
<Shell active="orders" go={go}>
  <window.Orders go={go} />
</Shell>
```
А карточка и мастер — на экранах `order-detail` и `order-new` (см. ваш `app.jsx`).

## 3. Эффект, который живёт НЕ здесь

Подсказка-расшифровка кодов разделов (наводите на чип `АР`, `КР`, `ЭОМ` —
всплывает полное название) реализована в **`app.jsx`**, а не в этом разделе.
В вашем проекте `app.jsx` остаётся как есть, поэтому подсказки продолжат работать.
Если будете править раздел в изоляции — учтите, что этот тултип придёт из `app.jsx`.

---

## 4. Проверка после переноса

- Список заявок: фильтры по типу/статусу/срочности, быстрые подсказки, поиск, переключатель вида (список/плитка).
- Карточка заявки: 5 вкладок, сайдбар «Заказчик» и «Требуются специалисты», превью-картинка объекта.
- Создание заявки: 5 шагов, живой предпросмотр справа со счётчиком заполнения.
- Если у карточек нет картинок — проверьте, что папка `assets/` на месте.
