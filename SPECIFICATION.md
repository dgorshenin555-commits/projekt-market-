# Спецификация платформы «Функция» (project-market)

> [!NOTE]
> Оригинальный документ ТЗ **не сохранился** в файлах проекта, Knowledge Items и логах прошлых бесед. Данная спецификация **восстановлена из изначального исходного кода** (коммит `1a22943` от 25.03.2026 и последующие коммиты).

---

## 1. Общее описание

**Функция** — единая платформа для проектирования, экспертизы и подбора решений в сфере строительства. Платформа объединяет:

- **Заказчиков** — публикуют заявки на проектирование
- **Проектировщиков** — откликаются на заявки, ведут проекты
- **Экспертов** — проводят экспертизу проектной документации
- **Производителей** — предлагают стройматериалы и решения

**Стек:** Next.js (App Router), TypeScript, vanilla CSS, localStorage (mock-данные).

---

## 2. Роли пользователей

| Роль | Код | Описание |
|------|-----|----------|
| Заказчик | `customer` | Создаёт заявки, выбирает исполнителей |
| Проектировщик | `designer` | Откликается на заявки, ведёт проекты |
| Эксперт | `expert` | Проводит экспертизу проектной документации |
| Производитель | `manufacturer` | Предлагает стройматериалы и решения |

## 3. Модули платформы

### 3.1 Лендинг (`/`)
- Hero-секция с описанием платформы и CTA-кнопками
- Карточка «Создание заявки» с описанием шагов (тип объекта → регион → стадия → разделы → файлы)
- Feature-карточки: Проектировщики, Экспертиза, Производители, Коммуникации, Рейтинги
- Тулбар снизу: CRM, EDI, Финансы, BIM, Маркет, Коммуникации, Производители
- Мобильное меню-сайдбар с навигацией по всем разделам

### 3.2 Авторизация (`/auth`)
- Вход по email/паролю
- Регистрация с выбором роли, указанием ФИО, компании, телефона, региона
- Хранение пользователей в `localStorage`

### 3.3 Дашборд (`/dashboard`)
- Обзорная панель после входа

### 3.4 Заявки на проектирование (`/orders`)

#### Список заявок (`/orders`)
- Фильтрация по типу объекта и статусу
- Отображение карточек заявок с ключевой информацией

#### Создание заявки (`/orders/new`)
- **Пошаговая форма:**
  1. Тип объекта (5 типов — см. ниже)
  2. Регион (16 вариантов)
  3. Стадия проектирования (П или РД) — *для частного строительства стадия пропускается*
  4. Выбор разделов и специалистов
  5. Масштаб проекта (один специалист / команда)
  6. Бюджет, сроки, описание

#### Детали заявки (`/orders/detail`)
- Полная информация о заявке
- Список откликов
- Возможность откликнуться (для проектировщиков)

#### Типы объектов

| Код | Название | Иконка |
|-----|---------|--------|
| `private` | Частное строительство | Кастомная PNG |
| `commercial` | Коммерческая недвижимость | 🏢 |
| `industrial` | Промышленность | 🏭 |
| `linear` | Линейные объекты | 🛤️ |
| `buildings` | Здания и сооружения | 🏗️ |

#### Стадии проектирования

- **Стадия П** — Проектная документация (17 разделов для капстроительства, 10 для линейных объектов)
- **Стадия РД** — Рабочая документация (7 групп: А-Ж, ~40 подразделов)

Каждый раздел содержит: код, название, список необходимых специалистов.

### 3.5 Проектировщики (`/designers`)
- Каталог проектировщиков (персоны и компании)
- Фильтрация по разделам и городу
- Карточки: имя, тип, разделы, город, СРО, рейтинг, отзывы, кол-во проектов

#### Профиль проектировщика (`/designers/[id]`)
- Расширенная информация
- Портфолио проектов
- Достижения
- Контактные данные

### 3.6 Эксперты (`/experts`)
- Каталог экспертов по экспертизе проектной документации
- Аналогичная структура каталогу проектировщиков

### 3.7 Экспертиза (`/expertise`)
- Трекер прохождения экспертизы
- Заявки на экспертизу (тип объекта, разделы, требование СРО)
- Статусы: «На проверке», «Ожидает исправлений», «Положительное заключение»
- Учёт замечаний: всего, исправлено, критических

### 3.8 Производители (`/manufacturers`)
- Каталог производителей стройматериалов
- Фильтрация по тэгам и городу
- Карточки: название, описание, тэги, рейтинг, кол-во проектов
- Slide-over модал с деталями производителя

#### Продукты производителя
- Название, описание, тэги, спецификации, количество сертификатов

### 3.9 Нормативы (`/standards`)
- Справочник нормативных документов
- Типы: ГОСТ, СП, СНиП, ТУ, ISO
- Статусы: Актуален, Устарел, Отменён
- Фильтрация по типу, секции, году
- Поддержка «избранных» документов

### 3.10 Коммуникации / Чат (`/chat`)
- Чаты между участниками проекта
- Обсуждение файлов (DWG, IFC, PDF)
- Адаптивный дизайн (мобильная версия)

### 3.11 Аналитика (`/analytics`)
- Дашборд с ключевыми метриками платформы

## 4. Модель данных

### Заявка (Order)
| Поле | Тип | Описание |
|------|-----|----------|
| id | string | Уникальный ID |
| customerId | string | ID заказчика |
| customerName | string | Имя заказчика |
| title | string | Название проекта |
| description | string | Описание |
| objectType | ObjectType | Тип объекта |
| region | string | Регион |
| scale | ProjectScale | Масштаб (один/команда) |
| stage | DesignStage | Стадия (П/РД) |
| sections | string[] | Выбранные разделы |
| specialists | string[] | Нужные специалисты |
| budget | string? | Бюджет |
| deadline | string? | Срок |
| status | OrderStatus | Статус заявки |
| responsesCount | number | Кол-во откликов |
| createdAt | string | Дата создания |

### Отклик (OrderResponse)
| Поле | Тип | Описание |
|------|-----|----------|
| id, orderId, designerId | string | Связи |
| message | string | Сообщение |
| proposedBudget | string? | Предложенный бюджет |
| proposedDeadline | string? | Предложенный срок |

### Проектировщик (Designer)
- id, name, type (person/company), sections, city, region, sroNumber, rating, reviewsLabel, projectsCount, yearsExperience, achievements, description

### Производитель (Manufacturer)
- id, name, description, tags, city, rating, projectsCount, deliveryRegion, website, email, phone

### Нормативный документ (StandardDocument)
- id, code, title, type (ГОСТ/СП/СНиП/ТУ/ISO), section, year, status, isFeatured

### Заявка на экспертизу (ExpertiseRequest)
- id, title, description, objectType, sections, requiredSro, budget, deadline, responsesCount

### Проект экспертизы (ExpertiseProject)
- id, title, company, status, totalRemarks, fixedRemarks, criticalRemarks, dueDate

## 6. Технические решения
- **Framework:** Next.js 15 (App Router, static export)
- **Язык:** TypeScript
- **Стилизация:** Vanilla CSS (globals.css ~67KB)
- **State management:** React Context + localStorage
- **Деплой:** GitHub Pages (static export с `trailingSlash`)
- **Иконки:** Lucide React
- **Шрифты:** Geist (через `next/font`)
