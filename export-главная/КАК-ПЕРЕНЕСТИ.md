# Перенос главной страницы в проект (VS Code)

Этот пакет содержит **главную страницу** платформы «Функция» целиком.
Цель: положить файлы в ваш проект и полностью заменить старую главную.

---

## 1. Что куда класть

Скопируйте файлы в корень проекта (поверх старых, с заменой):

### Обязательные — это и есть главная
| Файл | Назначение |
|------|-----------|
| `test_landing.jsx`   | Вся разметка и логика главной (компонент `TestLanding`) |
| `test_landing.css`   | Все стили главной (классы `tl-*`) |
| `landing_map_gl.jsx` | Секция «География» — интерактивная карта регионов (`RegionMap`) |
| `landing_map.css`    | Стили карты |

### Общие файлы — заменяйте, только если главная «поедет» (нет нужных иконок/цветов/шрифтов)
> Если ваш проект — та же база, эти файлы у вас уже такие же, и трогать их НЕ нужно.
> Замените их только если после переноса что-то выглядит сломанным.

| Файл | Назначение |
|------|-----------|
| `icons.jsx`       | Иконки `Icon` + логотип `FuncMark` (главная их использует) |
| `characters.jsx`  | Маскоты `Mascots` (фигурки в hero и блоке типов объектов) |
| `styles.css`      | Глобальные CSS-переменные: акцентные цвета, `--blue/--green/--amber/--pink`, цвета текста, шрифт Inter |
| `components.css`  | Импортируется из `styles.css` |
| `layouts.css`     | Импортируется из `styles.css` |
| `fonts/`          | Шрифт Inter (woff2) — кладётся в папку `fonts/` |

---

## 2. Подключение в HTML

Убедитесь, что в `<head>` есть ссылки на стили и шрифты:

```html
<!-- шрифты для главной -->
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link href="https://fonts.googleapis.com/css2?family=Manrope:wght@300..800&family=Space+Grotesk:wght@300..700&display=swap" rel="stylesheet" />
<link href="https://fonts.googleapis.com/css2?family=Poppins:wght@400;500;600;700&display=swap" rel="stylesheet" />

<!-- карта регионов (MapLibre) -->
<link rel="stylesheet" href="https://unpkg.com/maplibre-gl@4.7.1/dist/maplibre-gl.css" />

<!-- стили -->
<link rel="stylesheet" href="styles.css" />
<link rel="stylesheet" href="test_landing.css" />
<link rel="stylesheet" href="landing_map.css" />
```

И перед закрытием `<body>` — скрипты в таком порядке (главное: `icons` → `characters` → `landing_map_gl` → `test_landing`):

```html
<!-- библиотека карты -->
<script src="https://unpkg.com/maplibre-gl@4.7.1/dist/maplibre-gl.js"></script>

<script type="text/babel" src="icons.jsx"></script>
<script type="text/babel" src="characters.jsx"></script>
<script type="text/babel" src="landing_map_gl.jsx"></script>
<script type="text/babel" src="test_landing.jsx"></script>
```

Главная — это компонент `window.TestLanding`. В роутере он отображается на экране `landing`:

```jsx
if (screen === "landing") body = <window.TestLanding go={go} />;
```

## 3. Проверка после переноса

- Hero, кнопки «Разместить заявку» / «Найти заказы», поле-промпт.
- Блок «Возможности» — три карточки с анимацией чисел.
- «Как это работает» — слайдер из 3 шагов.
- «География» — карта регионов (если не появилась — проверьте подключение MapLibre).
- Тарифы, FAQ, футер.

Если карта не нужна — уберите из `test_landing.jsx` строку `<window.RegionMap go={go} />`
и тогда `landing_map_gl.jsx`, `landing_map.css` и MapLibre можно не подключать.
